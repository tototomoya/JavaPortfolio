
ユースケースは以下になります。

    1, お客様: 自販機を選び、お金を入れて、商品を選びます。

    2-1, 自販機: 商品の在庫と、小銭の更新を行います。

    2-2, 自販機: 不正パラメータの場合はトランザクションが失敗するので自販機の状態（内部の小銭）を元に戻します。

クラス構成は以下になります。

    Vending - パッケージ呼び出し時のインターフェースクラスです。

    User - ユーザクラスです。
    
    Location（A,B) - 地域ごとの自販機クラスです。（WEBアプリのようなステートレスなプログラムではないため、
                                                地域ごとのインスタンス生成にはシングルトンパターンを使用しています。)

        Money - 硬貨のクラスです。
        
        Item - 商品クラスです。
        
    Message（StaticClass） - ロギング、エラーレポートクラスです。
